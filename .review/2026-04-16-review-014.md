## 修复验证

### BUG-008: 倒计时不走 ✅（通过 debug log 验证）

**根因分析（多层问题）：**

1. **CGEventTap 是 placeholder** — `ActivityMonitor.startEventTap()` 从未实际安装 event tap，`recordActivity()` 从未被自动调用
2. **30 秒后 `isIdle = true`** — `checkIdleState()` 每 5 秒检查 `timeSinceLastActivity >= 30`，由于没有 CGEventTap 更新 `lastActivityTimestamp`，启动 30 秒后永远 idle
3. **BUG-007 的 `!wasIdle` guard** — `tick()` line 337 用 `!wasIdle` 阻止 elapsed 累加，idle 后倒计时停止
4. **`handleIdleDetected()` 重置所有 timer** — 即使 elapsed 能累加，每 30 秒 idle 检测会调用 `resetTimersAfterBreak()` 将 elapsed 清零

**Debug log 验证：**
```
修复前：
  elapsed_micro: 10→20→30→10（30 秒后被 handleIdleDetected 重置）

修复后：
  elapsed_micro: 10→20→30→40→50（持续增长，不受 idle 影响）
```

**修复方案（3 处改动）：**

| # | 改动 | 说明 |
|---|------|------|
| 1 | `!wasIdle` → `!isScreenLocked` | elapsed 累加只在屏幕锁定时暂停，不依赖 idle 检测 |
| 2 | idle 轮询改为检查 `isScreenLocked` | 只有屏幕锁定才触发 `handleIdleDetected()` + timer 重置 |
| 3 | 新增 `DistributedNotificationCenter` 监听 | BreakScheduler 直接监听 `screenIsLocked`/`screenIsUnlocked`，立即更新 `isScreenLocked` |

## 新问题

### MEDIUM

| ID | File | Issue |
|----|------|-------|
| M1 | `ActivityMonitor.swift` | `Unmanaged.passRetained(ActivityMonitorRef(...))` 创建了一个永远不会被释放的强引用（event tap 生命周期与 app 相同）。虽然不是 leak（app 退出即释放），但 `stopMonitoring()` 不会清理 event tap 和这个引用。如果需要支持动态启停，需要保存 tap 引用并在 stop 时 release |
| M2 | `BreakScheduler.swift` | `registerScreenLockObserver()` 和 `ActivityMonitor` 的 `ScreenLockObserver` 都监听同一通知，存在**双重响应**。BreakScheduler 直接设置 `isScreenLocked + wasIdle`，同时 5 秒轮询也检查 `activityMonitor.isScreenLocked`。两者可能产生竞争（通知先到 → 设置 wasIdle=true → 轮询看到 locked → 再次 handleIdleDetected）。`handleIdleDetected()` 内有 `guard !isBreakInProgress` 但没有 `guard !wasIdle` 防重入 |

### LOW

| ID | File | Issue |
|----|------|-------|
| L1 | `BreakScheduler.swift` | `screenLockObserverToken` 和 `screenUnlockObserverToken` 未在 deinit 中 remove。BreakScheduler 是 singleton 实际不影响，但不符合 observer 管理最佳实践 |
| L2 | `DashboardView.swift` | 窗口尺寸在两处定义：`DashboardView.frame(820×720)` 和 `DashboardWindowController.contentRect(840×740)`，差 20pt。应统一到一处常量 |
| L3 | `ActivityMonitor.swift` | CGEventTap callback 中每次事件都创建 `Task { await ref.monitor.recordActivity() }`。高频事件（如 mouseMoved）会创建大量 Task。可以添加节流（如每秒最多更新一次）|
