### 1.2 可测试性重构

**问题**：View 层大量 `SoundManager.shared`、`NotificationManager.shared` 直接调用，无法注入 mock。

**方案**：Environment-based DI for SwiftUI Views

```swift
// 新增 Environment Key
struct SoundPlayerKey: EnvironmentKey {
    static let defaultValue: any SoundPlaying = SoundManager.shared
}

extension EnvironmentValues {
    var soundPlayer: any SoundPlaying {
        get { self[SoundPlayerKey.self] }
        set { self[SoundPlayerKey.self] = newValue }
    }
}

// View 中使用
struct ExerciseSessionView: View {
    @Environment(\.soundPlayer) private var soundPlayer
    // 替代 SoundManager.shared.speak(...)
    // → soundPlayer.speak(...)
}

// 测试中注入
ExerciseSessionView(...)
    .environment(\.soundPlayer, MockSoundPlayer())
```

**需要提取 Environment Key 的 Singleton：**

| Singleton | 已有协议 | View 中直接调用 | Action |
|-----------|---------|----------------|--------|
| `SoundManager.shared` | ✅ `SoundPlaying` | ExerciseSessionView (6处) | 提取 EnvironmentKey |
| `NotificationManager.shared` | ✅ `NotificationSending` | MenuBarView (1处) | 提取 EnvironmentKey |
| `DashboardWindowController.shared` | ❌ | MenuBarView (1处) | 新建协议 + EnvironmentKey |
| `PreferencesWindowController.shared` | ❌ | MenuBarView (1处) | 新建协议 + EnvironmentKey |
| `ReportDataProvider.shared` | ✅ `ReportDataProviding` | MenuBarView (1处) | 提取 EnvironmentKey |
| `NightModeManager.shared` | ❌ | EyeGuardApp (1处) | 低优先级 |

## Sprint 2: 日志系统升级 (Structured Logging)

> 目标：从"有日志"到"日志可用于诊断、性能分析和用户行为洞察"

### 2.1 现状评估

**已有基础：** `Log` enum 用 `os.Logger` 定义了 14 个 category，95 处调用。

**不足：**

| 问题 | 说明 |
|------|------|
| **关键路径缺日志** | UI 交互（按钮点击、弹窗展示/关闭）几乎无日志 |
| **无结构化元数据** | `Log.scheduler.info("Timer started")` 只有文本，缺 breakType、duration 等结构化字段 |
| **无性能日志** | 不知道 tick() 耗时、UI 渲染延迟、TTS 延迟 |
| **无用户行为追踪** | 不知道用户多久跳过休息、做了几次操、偏好哪些设置 |
| **无日志级别策略** | 全是 `.info`，缺少 `.debug` / `.error` 的分层策略 |

### 2.2 日志级别策略

```
.debug   → 仅开发调试：tick 计数、动画帧、内部状态变化
.info    → 正常运行轨迹：休息开始/结束、眼保健操开始/完成
.notice  → 重要里程碑：日切换、首次启动、设置变更
.warning → 异常但可恢复：TTS 失败降级、权限缺失
.error   → 需要关注：数据保存失败、崩溃恢复
.fault   → 严重错误：数据损坏、不可恢复状态
```

### 2.3 需要补充日志的关键路径

```
用户行为日志 (Log.app / Log.scheduler):
  ├── 弹窗展示 → .info("Break notification shown", breakType, tier, sessionDuration)
  ├── 用户点击 Take Break → .info("Break taken", breakType, responseTime)
  ├── 用户点击 Skip → .notice("Break skipped", breakType, consecutiveSkips)
  ├── 用户点击 Postpone → .info("Break postponed", breakType, remainingPostpones)
  ├── 眼保健操开始 → .info("Exercise session started", exerciseCount, audioEnabled)
  ├── 眼保健操完成 → .info("Exercise session completed", duration, exercisesCompleted)
  ├── 眼保健操跳过 → .notice("Exercise session skipped", atExercise, elapsed)
  ├── Dashboard 打开 → .info("Dashboard opened")
  ├── Report 生成 → .info("Report generated", path)
  └── 设置变更 → .notice("Preference changed", key, oldValue, newValue)

性能日志 (Log.scheduler / Log.sound):
  ├── tick() 耗时 → .debug("Tick", elapsed: duration) // 仅当 > 100ms
  ├── TTS 延迟 → .debug("TTS latency", text, startToFirstWord)
  ├── Health score 计算 → .debug("Score calculated", score, breakdown, elapsed)
  └── 数据持久化 → .debug("Data saved", records, elapsed)

错误日志:
  ├── TTS 失败 → .warning("TTS unavailable, falling back to visual")
  ├── 数据读取失败 → .error("Failed to load history", error)
  ├── CGEventTap 权限 → .warning("Accessibility permission denied")
  └── 音频引擎失败 → .error("Audio engine failed to start", error)
```

### 2.4 结构化日志增强

利用 `os.Logger` 的插值功能，加入结构化字段：

```swift
// 改前
Log.scheduler.info("Break notification shown")

// 改后
Log.scheduler.info("""
    Break shown: type=\(breakType.rawValue, privacy: .public) \
    tier=\(tier.rawValue, privacy: .public) \
    sessionDuration=\(Int(sessionDuration))s \
    healthScore=\(healthScore)
    """)
```

### 2.5 诊断数据导出（P2 - 未来版本）

为了支持用户反馈问题时提供诊断信息：

```swift
/// 收集最近 N 条日志并导出为文本文件
actor DiagnosticsExporter {
    func exportRecentLogs(limit: Int = 500) async -> URL
    func exportSessionSummary() async -> String
}
```

- 加入 Preferences 中：「导出诊断数据」按钮
- 不包含隐私信息（os.Logger 的 `.private` 自动处理）

// 新增 Sources/App/DesignSystem.swift
enum EyeGuardDesign {
    // MARK: - Colors
    enum Colors {
        static let primary = Color.blue
        static let success = Color.green
        static let warning = Color.orange
        static let danger = Color.red
        static let accent = Color.teal          // 眼保健操主色
        
        static func healthScore(_ score: Int) -> Color { ... }
    }
    
    // MARK: - Spacing
    enum Spacing {
        static let xs: CGFloat = 4
        static let sm: CGFloat = 8
        static let md: CGFloat = 12
        static let lg: CGFloat = 16
        static let xl: CGFloat = 20
        static let xxl: CGFloat = 24

    // MARK: - Typography
    enum Typography {

        static let mono = Font.system(.body, design: .monospaced)

    // MARK: - Reusable Components
