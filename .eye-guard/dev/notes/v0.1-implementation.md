# v0.1 Implementation Notes

## Date
2026-04-14

## Overview
Initial project skeleton for EyeGuard ‚Äî a macOS menu bar app for eye health protection based on medical guidelines.

## Architecture

### App Layer (`EyeGuard/Sources/App/`)
- **EyeGuardApp.swift** ‚Äî `@main` entry point using SwiftUI `MenuBarExtra` (no dock icon)
- **AppDelegate.swift** ‚Äî System-level setup: accessibility permission check, monitoring startup, data directory creation
- **MenuBarView.swift** ‚Äî Menu bar popover UI with session timer, break controls, daily stats, and health score display

### Monitoring (`EyeGuard/Sources/Monitoring/`)
- **ActivityMonitor.swift** ‚Äî Actor-based activity monitor using CGEventTap (placeholder) with idle detection loop. Tracks last activity timestamp, determines idle state via 30-second threshold.

### Scheduling (`EyeGuard/Sources/Scheduling/`)
- **BreakType.swift** ‚Äî Enum defining micro (20-20-20), macro (hourly), and mandatory (2-hour) break types with intervals, durations, and display metadata
- **BreakScheduler.swift** ‚Äî `@Observable @MainActor` scheduler with 1-second tick loop. Tracks session duration, determines next break, records break events, and maintains health score.

### Notifications (`EyeGuard/Sources/Notifications/`)
- **NotificationManager.swift** ‚Äî Three-tier escalation system:
  - Tier 1: macOS `UNUserNotification` banner (gentle)
  - Tier 2: Semi-transparent overlay window (firm, placeholder)
  - Tier 3: Full-screen overlay requiring acknowledgment (mandatory, placeholder)

### Reporting (`EyeGuard/Sources/Reporting/`)
- **HealthScoreCalculator.swift** ‚Äî Calculates 0-100 health score with four weighted components: break compliance (40pts), continuous use discipline (30pts), screen time (20pts), break quality (10pts)
- **DailyReportGenerator.swift** ‚Äî Generates daily Markdown reports saved to `~/EyeGuard/reports/YYYY-MM-DD.md` with score breakdown, break details, and recommendations

### Models (`EyeGuard/Sources/Models/`)
- **Models.swift** ‚Äî Core data types: `BreakEvent`, `UsageSession` (immutable update pattern), `HealthScore` (clamped components), `UserPreferences` (with medical defaults), `DailyReport`

### Utils (`EyeGuard/Sources/Utils/`)
- **Constants.swift** ‚Äî All medical guideline constants (AAO, OSHA, EU Directive), escalation timing, health score weights, file paths

## Test Coverage (`EyeGuard/Tests/`)
- **EyeGuardTests.swift** ‚Äî Swift Testing suite covering:
  - BreakType values and codable round-trip
  - HealthScore clamping and total calculation
  - HealthScoreCalculator component calculations
  - BreakEvent codable round-trip
  - UsageSession immutable update patterns
  - UserPreferences default values matching medical guidelines

## Key Design Decisions
1. **Swift 6 strict concurrency** ‚Äî Actor for ActivityMonitor, @MainActor for BreakScheduler
2. **Immutable patterns** ‚Äî UsageSession uses `addingBreak()` / `ending()` returning new copies
3. **Value types** ‚Äî All models are structs (Sendable, Codable)
4. **Menu bar only** ‚Äî No dock icon, lives in MenuBarExtra
5. **Medical-evidence based** ‚Äî All timing constants sourced from AAO, OSHA, EU directives
