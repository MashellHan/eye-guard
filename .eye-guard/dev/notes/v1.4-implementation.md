# v1.4 Implementation Notes — Late Night Guardian (深夜提醒)

## Date: 2026-04-15

## Summary
Added a Late Night Guardian mode that activates after 10 PM (configurable), providing warmer, gentler reminders and tracking late-night usage statistics. The mascot transitions to sleeping state with amber-tinted speech bubbles and night-themed bilingual messages. Break reminders become more aggressive during night hours.

## Files Created
- `EyeGuard/Sources/Utils/NightModeManager.swift` — Observable singleton managing night mode state, configurable start/end hours, night messages, screen time tracking
- `EyeGuard/Tests/NightModeManagerTests.swift` — 13 tests covering messages, configuration, formatting, and state

## Files Modified
- `EyeGuard/Sources/Mascot/SpeechBubbleView.swift` — Added `isNightMode` parameter for amber-tinted warm colors (background, text, shadow)
- `EyeGuard/Sources/Mascot/MascotContainerView.swift` — Integrated NightModeManager into state sync loop; night screen time display every 15 min; night-specific break messages; replaced hardcoded night check with NightModeManager
- `EyeGuard/Sources/App/EyeGuardApp.swift` — MenuBarLabel shows 🌙 prefix when night mode active
- `EyeGuard/Sources/App/PreferencesView.swift` — Added "Night Mode 深夜提醒" section with start hour slider (20-23)
- `EyeGuard/Sources/Utils/Logging.swift` — Added nightMode and colorAnalysis loggers

## Night Mode Features
| Feature | Description |
|---------|-------------|
| Auto-activation | Activates after configurable hour (default 22:00) |
| Sleeping mascot | Transitions to `.sleeping` state with zzz animation |
| Warm colors | Speech bubbles use amber tint (warm beige background, brown text) |
| Night messages | 8 unique night-themed bilingual messages |
| Night break messages | 3 more urgent break-specific night messages |
| Screen time tracking | Tracks and displays late-night screen time every 15 min |
| Menu bar indicator | Shows 🌙 emoji in menu bar during night mode |
| Configurable | Night start hour adjustable in Preferences (20-23) |
| Break multiplier | 0.5x interval (breaks come twice as often at night) |

## Architecture
- `NightModeManager` is a `@MainActor @Observable` singleton for reactive UI updates
- Static message arrays are `nonisolated` for safe access from test contexts
- Night mode state is polled every 2 seconds in the existing `startStateSync()` tick loop
- Replaces the previous hardcoded `hour >= 22 || hour < 6` check in `updateMascotState()`
- Configurable via `@AppStorage` keys: `nightModeStartHour`, `nightModeEndHour`
