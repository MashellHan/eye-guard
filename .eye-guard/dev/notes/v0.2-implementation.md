# v0.2 Implementation Notes

## Overview

v0.2 wires up the complete activity ‚Üí scheduler ‚Üí notification pipeline, fixes 7 bugs from Tester reports, and adds comprehensive test coverage.

## Changes by Category

### Lead Review Fixes

| ID | Description | Files Changed |
|----|-------------|---------------|
| C1 | BreakScheduler timerLoop concurrency fix ‚Äî removed `[weak self]` and `await MainActor.run` | `BreakScheduler.swift` |
| H1 | Wire ActivityMonitor ‚Üí BreakScheduler via polling in `tick()` with `wasIdle` tracking | `BreakScheduler.swift` |
| H2 | Wire BreakScheduler ‚Üí NotificationManager with `onTaken`/`onSkipped` callbacks | `BreakScheduler.swift` |
| H4 | NotificationManager stores and invokes callbacks; `onTaken` in `acknowledgeBreak()`, `onSkipped` on escalation timeout | `NotificationManager.swift` |
| H5 | Per-break-type timer reset logic: micro resets micro only, macro resets macro+micro, mandatory resets all | `BreakScheduler.swift` |
| H6 | Protocols `ActivityMonitoring` and `NotificationSending` for testability; BreakScheduler accepts via init | `Protocols/*.swift`, `BreakScheduler.swift`, `ActivityMonitor.swift`, `NotificationManager.swift` |

### Bug Fixes

| ID | Description | Root Cause | Fix |
|----|-------------|------------|-----|
| BUG-001 | resetTimersAfterBreak was no-op | Empty switch cases | Per-type reset with `elapsedPerType` dict |
| BUG-002 | Health score used session not daily totals | `currentSessionDuration` used instead of daily accumulator | Added `totalScreenTimeToday` with delta accumulation in `tick()` |
| BUG-003 | checkForDueBreaks timing race / double-fire | Modular arithmetic edge case | Added `lastNotifiedCycle` per-type dedup guard |
| BUG-004 | HealthScoreCalculator boundary arithmetic | Discontinuity at recommended max boundary | Rewrote `calculateScreenTimeScore()` with smooth linear interpolation |
| BUG-005 | ActivityMonitor no reset mechanism | No public reset method | Added `resetState()` method to actor and protocol |
| BUG-006 | Snooze doesn't reschedule | `snooze()` dismissed without rescheduling | Implemented snooze with `Task.sleep` and callback |
| BUG-007 | DateFormatter timezone | No explicit timezone set | Set `.timeZone = .current` on all DateFormatters |

### Additional v0.2 Features

- Replaced ALL `print()` statements with `os.Logger` via centralized `Log` enum
- Made `DailyReportGenerator.generate()` async for background file I/O
- Moved NotificationManager permission request out of `init` into explicit `setup()`
- Fixed `UsageSession.endTime`, `activeTime`, `breaks` from `var` to `let`
- Extracted shared `TimeFormatting.formatDuration()` and `formatTimerDisplay()` to `Utils/TimeFormatting.swift`
- Added daily rollover logic in both `BreakScheduler` and `ActivityMonitor` (reset at midnight)

### Test Framework Fix

- Added `swift-testing` (0.12.0+) as a package dependency in `Package.swift` to resolve `no such module 'Testing'` error in Command Line Tools environments
- All tests use Swift Testing framework (`import Testing`, `@Test`, `#expect`, `@Suite`)
- The Command Line Tools SDK does not ship `Testing` or `XCTest` as built-in importable modules; the `swift-testing` package provides the `Testing` module

### Test Coverage

Added 4 new test files with 65 total tests (up from ~15):

| File | Tests | Coverage |
|------|-------|----------|
| `EyeGuardTests.swift` | 13 | Models, BreakType, HealthScore, UsageSession, UserPreferences |
| `BreakSchedulerTests.swift` | 13 | Timer logic, break selection, reset behavior, idle handling |
| `HealthScoreCalculatorTests.swift` | 17 | Boundary cases, monotonicity, mixed scenarios |
| `DailyReportGeneratorTests.swift` | 5 | Format validation, empty data, health score integration |
| `NotificationManagerTests.swift` | 17 | Mock verification, TimeFormatting, ActivityMonitor mock |
