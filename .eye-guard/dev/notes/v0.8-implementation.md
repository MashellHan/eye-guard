# v0.8 Implementation Notes — Preferences UI (SwiftUI Settings Window)

## Date: 2026-04-15

## Overview
Added a full SwiftUI preferences window accessible from the menu bar, allowing users to customize break intervals, durations, notification behavior, and report settings.

## Changes Made

### 1. PreferencesView.swift (new)
- **TabView** with 4 tabs: Breaks, Notifications, General, Reports
- **Breaks tab**:
  - Micro Break: interval slider (10-30 min), duration slider (10-60 sec)
  - Macro Break: interval slider (30-90 min), duration slider (3-15 min)
  - Mandatory Break: interval slider (60-180 min), duration slider (10-30 min)
  - "Reset to Defaults" button
  - Each slider shows current value, range labels, and medical guideline description
- **Notifications tab**:
  - Toggle for Tier 1 (gentle notification), Tier 2 (floating overlay), Tier 3 (full-screen)
  - Each toggle has description text explaining what it does
  - Notification sound on/off toggle
- **General tab**:
  - Launch at Login toggle (persisted, SMAppService integration placeholder)
  - Show in Dock toggle
  - About section with version info
- **Reports tab**:
  - Auto-generate daily report toggle
  - Report directory display with "Open in Finder" button
- All values persisted via `@AppStorage` (UserDefaults)
- Grouped form style for native macOS appearance

### 2. PreferencesWindowController.swift (new)
- Singleton `NSWindow` manager for the preferences window
- Creates `NSHostingController` wrapping `PreferencesView`
- Floating window level (appears above menu bar popover)
- Reuses existing window if already open

### 3. UserPreferencesManager.swift (new)
- Enum with static methods to bridge `@AppStorage` → `UserPreferences` struct
- `load()`: Reads UserDefaults values, converts units (minutes → seconds), falls back to defaults
- `isAutoReportEnabled`: Checks auto-report preference
- `isLaunchAtLoginEnabled`: Checks launch-at-login preference

### 4. MenuBarView.swift
- "Preferences..." button now calls `PreferencesWindowController.shared.showPreferences()`
- Removed TODO comment

### 5. build-app.sh
- Updated version to 0.8.0, build version to 8

### 6. Tests (PreferencesTests.swift — new)
- 10 test cases:
  - Default values when no UserDefaults set
  - Custom micro interval from UserDefaults
  - Custom micro duration from UserDefaults
  - Custom sound setting
  - Auto report default (true)
  - Auto report custom value
  - Launch at login default (false)
  - Launch at login custom value
  - All break types enabled
  - Escalation always enabled

## Architecture Decisions
- **@AppStorage over manual UserDefaults**: SwiftUI-native persistence for automatic view updates
- **Separate UserPreferencesManager**: Clean bridge between SwiftUI @AppStorage and value-type UserPreferences
- **NSWindow over Settings scene**: MenuBarExtra apps can't use SwiftUI Settings scene, so manual NSWindow management
- **Floating window level**: Ensures preferences appear above the menu bar popover
- **Unit conversion in manager**: PreferencesView stores human-readable units (minutes), manager converts to seconds
