# v0.3 Implementation Notes

## Problem

EyeGuard crashed on launch because Swift Package Manager builds a bare executable, not a macOS `.app` bundle. `UNUserNotificationCenter` and `MenuBarExtra` require a proper app bundle with `Info.plist` and a `CFBundleIdentifier`.

Error: `bundleProxyForCurrentProcess is nil: mainBundle.bundleURL file:///.../release/`

## Changes

### 1. App Bundle Infrastructure (`scripts/build-app.sh`)

Created a shell script that packages the SPM-built binary into a proper macOS `.app` bundle:

```
EyeGuard.app/
├── Contents/
│   ├── Info.plist        # Bundle metadata
│   ├── MacOS/
│   │   └── EyeGuard     # Executable (copied from .build/release/)
│   └── Resources/       # Empty for now (icons, assets later)
```

Key Info.plist entries:
- `CFBundleIdentifier`: `com.eyeguard.app` — required for `UNUserNotificationCenter`
- `LSUIElement`: `true` — menu-bar-only app (no Dock icon)
- `NSAppleEventsUsageDescription` — system event monitoring description

### 2. NotificationManager Safety Guards

Added `Bundle.main.bundleIdentifier` guards to prevent crashes when running outside an app bundle (e.g., during `swift test` or direct binary execution):

- `requestNotificationPermission()` — skips if no bundle
- `sendTier1Notification()` — skips if no bundle
- `dismissAllOverlays()` — only calls `UNUserNotificationCenter` if bundle exists

Similarly guarded `AppDelegate.setupNotifications()`.

This means the app degrades gracefully: it runs without notifications in bare-executable mode and with full notifications in `.app` bundle mode.

### 3. Menu Bar Countdown Display

Updated `EyeGuardApp.swift` to show a live countdown to the next break in the menu bar:

- Format: `20:00` → `19:59` → ... → `00:00` (MM:SS countdown)
- Shows `⏸` when paused
- Uses the eye icon (`eye.trianglebadge.exclamationmark`) as the SF Symbol
- Extracted into a `MenuBarLabel` view for clean separation

The countdown is driven reactively by `BreakScheduler.timeUntilNextBreak` (updated every second by the existing timer loop).

## How to Build and Run

```bash
# 1. Build the release binary
swift build -c release

# 2. Create the .app bundle (also rebuilds if needed)
chmod +x scripts/build-app.sh
bash scripts/build-app.sh

# 3. Launch
open EyeGuard.app

# 4. Kill
pkill -f EyeGuard.app
```

## Testing

All 65 tests pass with `swift test`. The bundle guards ensure tests work correctly without requiring a real app bundle.
