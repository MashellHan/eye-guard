# v0.5 Implementation Notes

## Features

### 1. Full-Screen Break Overlay (Tier 3)

**File**: `EyeGuard/Sources/Notifications/FullScreenOverlayView.swift`

A SwiftUI view that covers the entire screen when the user has been using the computer for 120+ minutes continuously without a proper break.

**Visual design:**
- Semi-transparent dark background (0.7 opacity) with `NSVisualEffectView` blur
- Large eye SF Symbol (`eye.trianglebadge.exclamationmark`, 80pt) with pulsing animation
- Warning title and subtitle explaining the situation
- 15-minute countdown displayed in a circular progress ring (160pt diameter)
- Pulsing glow effect on the countdown number when active
- Health score display with color-coded heart icon (green/orange/red)
- Random medical tip from a pool of 12 evidence-based tips
- Two buttons:
  - **"Take 15-min Break"** (primary, green) — starts the countdown and locks the overlay
  - **"I Need 5 More Minutes"** (secondary, small) — adds 5 minutes extension, max 2 uses

**Window behavior:**
- Window level: `.screenSaver` (above everything including other apps)
- Covers ALL connected screens (multi-monitor support)
- Cannot be easily dismissed (no close button, no title bar)
- Smooth 0.5s fade-in animation
- Cmd+Q still works for emergency exit

### 2. OverlayWindowController Updates

**File**: `EyeGuard/Sources/Notifications/OverlayWindow.swift`

Added multi-monitor full-screen overlay management:

```swift
func showFullScreenOverlay(healthScore: Int, onTaken: @escaping @Sendable () -> Void)
func dismissFullScreen()
var isFullScreenShowing: Bool
```

- Creates one `NSWindow` per connected `NSScreen`
- Each window is positioned to cover its screen's full frame
- Tracks windows in `fullScreenWindows` array (separate from Tier 2 `window`)
- Dismisses all screens simultaneously with fade-out animation

### 3. Continuous Use Tracking

**File**: `EyeGuard/Sources/Scheduling/BreakScheduler.swift`

New tracked properties:
- `longestContinuousSession: TimeInterval` — updated on each tick, tracks the longest unbroken usage stretch today
- `continuousUseWarnings: Int` — counts how many times Tier 3 has been triggered today

New method:
- `checkContinuousUse()` — called every tick, triggers Tier 3 mandatory break notification when continuous use reaches 120 minutes. Uses cycle-based tracking to prevent double-firing (similar to `lastNotifiedCycle` for regular breaks).

Reset behavior:
- `resetDaily()` now also resets `longestContinuousSession` and `continuousUseWarnings`
- `recalculateHealthScore()` now uses `longestContinuousSession` instead of `currentSessionDuration` for more accurate discipline scoring

### 4. NotificationManager Tier 3 Wiring

**File**: `EyeGuard/Sources/Notifications/NotificationManager.swift`

Replaced the `showTier3Fullscreen` TODO stub with a real implementation:

```swift
private func escalateToTier3(healthScore: Int, onTaken: @escaping @Sendable () -> Void) {
    overlayController.dismiss()  // Dismiss Tier 2 first
    overlayController.showFullScreenOverlay(healthScore: healthScore, onTaken: onTaken)
}
```

- `dismissAllOverlays()` now also calls `overlayController.dismissFullScreen()`
- Tier 3 escalation properly chains from Tier 2 timeout for mandatory breaks

## Architecture Decisions

1. **Separate window tracking**: `fullScreenWindows: [NSWindow]` is tracked separately from the Tier 2 `window: NSWindow?` to avoid interference between tiers.

2. **NSVisualEffectView wrapper**: Created `VisualEffectBlur` as an `NSViewRepresentable` to provide native macOS blur behind the overlay.

3. **Extension limit**: The "5 More Minutes" button is limited to 2 uses per overlay appearance, preventing indefinite postponement.

4. **Medical tip pool**: 12 evidence-based eye health tips are randomly selected and remain stable during a single overlay appearance (using `@State`).

5. **Cycle-based warning**: `continuousUseWarnings` uses integer cycle counting (`Int(duration / threshold)`) to prevent double-firing, matching the pattern used in `checkForDueBreaks`.
