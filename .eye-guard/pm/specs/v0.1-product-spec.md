## 1. Product Vision

**EyeGuard** is a medical-grade eye health guardian for macOS that monitors screen time and reminds users to take breaks based on ophthalmological evidence.

The core principle is the **20-20-20 rule**, recommended by the American Academy of Ophthalmology (AAO): every 20 minutes, look at something 20 feet away for at least 20 seconds. This simple practice significantly reduces digital eye strain (computer vision syndrome), which affects an estimated 50-90% of computer workers.

EyeGuard goes beyond a simple timer. It uses intelligent activity detection to track actual screen engagement, calculates a daily eye health score, and generates actionable reports. The notification system escalates progressively, respecting the user's workflow while firmly protecting their eye health.

### Design Principles

1. **Evidence-based**: Every default value is grounded in ophthalmological research
2. **Respectful**: Gentle by default, assertive only when necessary
3. **Intelligent**: Detects idle time; doesn't nag when you're already away
4. **Transparent**: Daily reports show exactly what happened and why
5. **Native**: Pure macOS experience, no Electron, minimal resource usage

## 3. Core Features (v0.1 MVP)

### 3.1 Menu Bar App

- **Status icon**: Eye icon (👁) in the macOS menu bar
- **Timer display**: Shows time until next break in the menu bar (e.g., `17:42`)
- **Popover**: Click to reveal session summary, quick controls, and current health score
- **States**:
  - 🟢 Green: Healthy usage pattern
  - 🟡 Yellow: Approaching break time
  - 🔴 Red: Overdue for a break or continuous use exceeded

### 3.2 Activity Monitoring

- **Method**: `CGEventTap` for keyboard and mouse event monitoring
- **Events tracked**: Key presses, mouse moves, mouse clicks, scroll events
- **Privacy**: No keystrokes are recorded. Only event timestamps are captured.
- **Permissions**: Requires macOS Accessibility permission (System Settings > Privacy & Security > Accessibility)
- **Fallback**: If Accessibility permission is denied, fall back to `IOHIDManager` or `NSEvent.addGlobalMonitorForEvents` with reduced fidelity

### 3.3 Idle Detection

- **Threshold**: No keyboard/mouse input for 30+ seconds = idle state
- **Behavior when idle**:
  - Pause the break timer (user is already resting)
  - If idle duration >= break duration required, count it as a break taken
  - Resume timer when activity resumes
- **Extended idle** (5+ minutes): Treat as session end. Start fresh timer on resume.

### 3.4 Micro-Break Timer (20-20-20 Rule)

- **Interval**: Every 20 minutes of active screen time
- **Break duration**: 20 seconds
- **Action**: Look at something 20 feet (6 meters) away
- **Notification**: Tier 1 (gentle) by default
- **Skip**: User can skip/snooze (but it affects health score)
- **Auto-dismiss**: If user goes idle for 20+ seconds during the notification, auto-dismiss and count as break taken

### 3.5 Macro-Break Timer

- **Interval**: Every 60 minutes of active screen time
- **Break duration**: 5-10 minutes (configurable, default 5 min)
- **Action**: Stand up, stretch, walk around, hydrate
- **Notification**: Tier 1 first, escalates to Tier 2 if ignored
- **Relationship to micro-breaks**: Macro-break resets the micro-break timer

### 3.6 Maximum Continuous Use Limit

- **Threshold**: 120 minutes of continuous active screen time without a macro-break
- **Action**: Triggers Tier 3 (urgent) notification
- **Behavior**: Full-screen overlay with health warning. Cannot be dismissed for 30 seconds (configurable).
- **Medical basis**: Prolonged uninterrupted screen time is associated with increased risk of digital eye strain, dry eye, and musculoskeletal issues (AOA guidelines)

### 3.7 Basic Notifications

- **Channel**: macOS Notification Center (`UNUserNotificationCenter`)
- **Sound**: Gentle chime (optional, configurable)
- **Actions**: "Take Break", "Snooze 5 min", "Skip"
- **Do Not Disturb**: Respect macOS Focus modes. Queue notifications and deliver when Focus ends.

### 3.8 Simple Preferences

| Setting | Default | Range |
|---------|---------|-------|
| Micro-break interval | 20 min | 10-30 min |
| Micro-break duration | 20 sec | 10-60 sec |
| Macro-break interval | 60 min | 30-90 min |
| Macro-break duration | 5 min | 3-15 min |
| Max continuous use | 120 min | 60-180 min |
| Idle threshold | 30 sec | 15-120 sec |
| Launch at login | Off | On/Off |
| Notification sound | On | On/Off |
| Enable micro-breaks | On | On/Off |
| Enable macro-breaks | On | On/Off |

## 4. Notification Escalation UX Design

### 4.1 Escalation Flow

```
Active screen time reaches interval
         │
         ▼
   ┌─────────────┐
   │   Tier 1    │  macOS notification banner
   │   Gentle    │  "Time for a 20-second eye break!"
   └──────┬──────┘
          │ User ignores 2 consecutive Tier 1 notifications
          ▼
   ┌─────────────┐
   │   Tier 2    │  Floating translucent overlay window
   │  Assertive  │  Top-right corner, countdown timer
   └──────┬──────┘
          │ Continuous use exceeds 120 minutes
          ▼
   ┌─────────────┐
   │   Tier 3    │  Full-screen semi-transparent overlay
   │   Urgent    │  Blur effect, health warning, forced pause
   └─────────────┘
```

### 4.2 Tier 1 - Gentle Notification

- **Type**: macOS Notification Center banner
- **Appearance**: Standard system notification with EyeGuard icon
- **Title**: "Eye Break Time 👀"
- **Body**: "Time for a 20-second eye break! Look at something 20 feet away."
- **Actions**: "Start Break" / "Snooze 5 min"
- **Auto-dismiss**: After 15 seconds if no interaction
- **Frequency**: Every micro-break interval (default 20 min)

### 4.3 Tier 2 - Assertive Overlay

- **Type**: `NSPanel` floating window, `NSWindow.Level.floating`
- **Position**: Top-right corner of the primary display, 20px margin
- **Size**: 320 x 180 points
- **Appearance**: Translucent background (`NSVisualEffectView` with `.hudWindow` material), rounded corners (16pt)
- **Content**:
  - Icon: Eye emoji or custom icon
  - Title: "You've been working for X minutes"
  - Subtitle: "Take a break to protect your eyes"
  - Countdown timer: Large text showing remaining break time
  - Progress ring: Visual timer around the countdown
  - Button: "I'm taking a break" (dismisses and logs compliance)
- **Behavior**:
  - Stays on top of all windows
  - Cannot be closed during countdown (only "I'm taking a break" button)
  - Semi-transparent so user can still see their work
  - Fades in with 0.3s animation
- **Trigger**: User ignored 2 consecutive Tier 1 notifications

### 4.4 Tier 3 - Urgent Full-Screen Overlay

- **Type**: `NSWindow` covering full screen, `NSWindow.Level.screenSaver`
- **Appearance**: Semi-transparent dark overlay (60% opacity) with Gaussian blur on background content
- **Content**:
  - Large countdown timer (center screen)
  - Health warning icon (⚠️)
  - Message: "You've been using the screen continuously for over 2 hours."
  - Subtext: "Prolonged screen time increases risk of digital eye strain, dry eye, and headaches. Please take a 5-minute break."
  - Break suggestions: "Look away from screen", "Stand and stretch", "Hydrate"
  - Progress bar: Shows mandatory wait time (30 seconds minimum)
  - "Start Break" button: Enabled after mandatory wait period
- **Behavior**:
  - Covers entire screen including menu bar
  - Cannot be dismissed for 30 seconds (configurable)
  - After 30 seconds, "Dismiss" button appears (but "Start Full Break" is primary)
  - Desktop content blurred behind overlay
  - Escape key does NOT dismiss during mandatory period
- **Trigger**: Continuous active screen time exceeds 120 minutes without a macro-break

### 4.5 Escalation Reset

- Taking any break (micro or macro) resets the escalation tier back to Tier 1
- Idle detection counting as a break also resets escalation
- Snoozing does NOT reset escalation (only delays next notification)

## 5. Health Score Algorithm (0-100)

### 5.1 Score Components

```
Health Score = Break Compliance (40) + Continuous Use (30) + Screen Time (20) + Break Quality (10)
```

### 5.2 Break Compliance (40 points)

Measures the percentage of suggested breaks that the user actually took.

```
compliance_ratio = breaks_taken / breaks_suggested
points = compliance_ratio * 40

Examples:
  - 10/10 breaks taken → 40 points
  - 7/10 breaks taken  → 28 points
  - 3/10 breaks taken  → 12 points
```

- A break is "taken" if the user was idle for >= the suggested break duration within 2 minutes of the notification
- Snoozed-then-taken breaks count at 80% value (penalize delay)
- Skipped breaks count as 0

### 5.3 Continuous Use Discipline (30 points)

Penalizes stretches of continuous active screen time exceeding 40 minutes without any break.

```
penalty_per_violation = 5 points
max_penalty = 30 points

For each continuous stretch > 40 min:
  penalty += min(5, (stretch_minutes - 40) / 10 * 2)

points = max(0, 30 - total_penalty)

Examples:
  - No stretch > 40 min         → 30 points
  - One 50-min stretch          → 28 points (penalty: 2)
  - One 90-min stretch          → 20 points (penalty: 10)
  - Three 60-min stretches      → 18 points (penalty: 12)
```

### 5.4 Total Screen Time (20 points)

Based on daily total active screen time vs. recommended maximum.

```
Recommended max: 8 hours of active screen time (occupational guideline)
Ideal: < 6 hours

points = 20                           if total <= 6h
points = 20 - ((total - 6) * 5)      if 6h < total <= 10h
points = 0                            if total > 10h

Examples:
  - 5 hours total  → 20 points
  - 7 hours total  → 15 points
  - 9 hours total  → 5 points
  - 11 hours total → 0 points
```

### 5.5 Break Quality (10 points)

Measures whether the user actually looked away from the screen during breaks vs. just dismissing the notification.

```
quality_ratio = genuine_breaks / total_breaks_taken

A break is "genuine" if:
  - No keyboard/mouse activity detected during the break period
  - Break lasted at least 80% of the suggested duration

points = quality_ratio * 10

Examples:
  - All breaks had no activity → 10 points
  - Half the breaks had activity during them → 5 points
```

### 5.6 Score Interpretation

| Score Range | Rating | Color | Description |
|-------------|--------|-------|-------------|
| 90-100 | Excellent | 🟢 Green | Outstanding eye care habits |
| 70-89 | Good | 🟢 Green | Healthy with minor room for improvement |
| 50-69 | Fair | 🟡 Yellow | Needs attention, some risky patterns |
| 30-49 | Poor | 🟠 Orange | Significant risk of eye strain |
| 0-29 | Critical | 🔴 Red | High risk, immediate habit change needed |

### 8.2 Key Dependencies

- **Framework**: SwiftUI + AppKit (hybrid for menu bar and overlays)
- **Persistence**: SwiftData (macOS 14+)
- **Notifications**: UserNotifications framework
- **Activity monitoring**: CoreGraphics (CGEventTap)
- **Login item**: ServiceManagement (SMAppService)
- **No third-party dependencies** for v0.1-v0.6

### 8.3 Privacy & Security

- No network access required (fully offline)
- No keystroke content is recorded (only event timestamps)
- All data stored locally in app sandbox
- Accessibility permission required (explained in onboarding)
- No analytics or telemetry in v1.0

## 10. Open Questions

1. **App Store vs. direct distribution**: App Store limits CGEventTap usage. May need to distribute via DMG with notarization.
2. **Sandboxing**: CGEventTap requires non-sandboxed app or temporary entitlement exception. Need to investigate App Store compatibility.
3. **Multi-monitor**: Should Tier 3 overlay cover all screens or just the primary? (Recommendation: all screens)
4. **Calendar integration**: Should we detect meetings (calendar events) and suppress notifications? (Deferred to post-v1.0)
5. **Pomodoro mode**: Some users may prefer Pomodoro-style work/break cycles. Consider as alternative mode in v1.0+.
