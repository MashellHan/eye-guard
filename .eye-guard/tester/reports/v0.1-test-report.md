## Recommendations for v0.2

### Priority 1: CRITICAL Fixes

1. **Fix `resetTimersAfterBreak` no-op** (BUG-001): The switch statement does nothing for any case. This means after taking a macro break, micro-break timers are not independently reset - only the full session resets. This defeats the purpose of hierarchical break scheduling.

2. **Fix `recalculateHealthScore` daily tracking** (BUG-002): The health score only considers the current session's screen time, not the full day's accumulated time. After idle-resume cycles, the longest session and total screen time reset.

### Priority 2: HIGH - Test Coverage

3. **Add BreakScheduler tests**: This is the core logic of the app with 0% coverage. Use dependency injection for the timer mechanism to make it testable without real `Task.sleep` delays.

4. **Add HealthScoreCalculator boundary tests**: The screen time scoring has a potential arithmetic issue at the recommended max boundary (BUG-004).

5. **Add DailyReportGenerator tests**: Report generation is untested and writes to the filesystem. Inject the file manager to make it testable.

### Priority 3: MEDIUM - Architecture

6. **Make NotificationManager testable**: Extract protocol for notification delivery, inject into BreakScheduler. The current singleton + private init pattern prevents testing.

7. **Make ActivityMonitor testable**: The actor singleton with CGEventTap dependency cannot be unit tested. Extract an `ActivityMonitoring` protocol.

8. **Add daily rollover logic**: There is no mechanism to reset daily counters at midnight. `breaksTakenToday`, `breaksSkippedToday`, and `todayBreakEvents` persist indefinitely.

9. **Implement snooze rescheduling** (BUG-006): The `snooze()` method cancels escalation but never reschedules the notification.

### Priority 4: LOW - Polish

10. **Set explicit timezone on DateFormatter** (BUG-007): Avoid locale-dependent date string formatting in `DailyReport`.

11. **Replace `print()` with `os.Logger`**: All logging uses `print()` which is not appropriate for production macOS apps.

12. **Implement CGEventTap**: The ActivityMonitor has a placeholder - idle detection cannot actually detect user input without it.

### Architecture Recommendations

- **Dependency Injection**: `BreakScheduler` directly instantiates `HealthScoreCalculator`. Inject it via protocol for testability.
- **Protocol extraction**: `NotificationManager`, `ActivityMonitor`, and `DailyReportGenerator` should conform to protocols so BreakScheduler can work with mocks in tests.
- **Clock abstraction**: Inject a `Clock` protocol (or use Swift's `ContinuousClock`) so timer-based logic can be tested without real delays.
