## Test Runtime Analysis

**Notable**: Test runtime jumped from ~0.055s (v0.2) to **17.295s** (v0.4).

The `OverlayWindowController` tests and `BreakScheduler` tests now take 12-17 seconds each. This is caused by:

1. **`dismissOverlay` test**: Uses `try await Task.sleep(for: .milliseconds(500))` to wait for the 0.3s fade-out animation. This test alone accounts for ~5s overhead (animation context setup in test environment).
2. **NSWindow creation in tests**: Creating real `NSWindow` + `NSHostingView` instances in test context is slower than pure model tests. Each overlay test creates actual AppKit windows.
3. **BreakScheduler timer loop**: The `startTimerLoop()` Task runs during tests, adding background work.

**Recommendation**: Consider a `OverlayWindowControlling` protocol with a mock for faster unit tests. Reserve real-window tests for integration suite.

## Overlay Test Quality Assessment

### Strengths

1. **State machine coverage**: Tests verify the full lifecycle — initial state, show, dismiss, replace, no-op dismiss. This catches regressions in the window management logic.

2. **All break types tested**: The `allBreakTypes` test iterates `BreakType.allCases`, ensuring no case is missed in `showBreakOverlay`.

3. **Animation-aware dismiss test**: The `dismissOverlay` test correctly waits 500ms for the 0.3s fade-out animation to complete before checking `isShowing`. This avoids false failures from animation timing.

4. **Replace semantics verified**: The `replaceOverlay` test confirms that calling `showBreakOverlay` while one is already showing replaces (not stacks) the overlay.

### Weaknesses

1. **No full-screen overlay tests**: The Tier 3 overlay path (`showFullScreenOverlay`, `dismissFullScreen`, `isFullScreenShowing`) has zero test coverage. This is the most complex window management code.

2. **No callback verification**: Tests pass empty closures `{}` for `onTaken` and `onSkipped`. No test verifies that callbacks are actually invoked when expected.

3. **No view content verification**: Tests only check `isShowing` state. No verification that the correct break type is displayed, countdown works, or buttons are functional.

4. **No concurrent show/dismiss test**: The dismiss-animation-then-show race condition (M-V04-1 in code review) has no test coverage.
