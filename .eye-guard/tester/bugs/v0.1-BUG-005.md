# BUG-005: `ActivityMonitor` singleton actor cannot reset state for daily rollover or testing

**Bug ID**: v0.1-BUG-005
**Title**: ActivityMonitor singleton has no state reset mechanism
**Severity**: MEDIUM
**Component**: ActivityMonitor
**File**: `EyeGuard/Sources/Monitoring/ActivityMonitor.swift`

## Steps to Reproduce

1. Start the app and use it all day
2. Leave it running overnight
3. The next morning, `ActivityMonitor.shared` still holds yesterday's state
4. There is no `reset()` method and no daily rollover logic
5. Additionally, `startMonitoring()` has an `isMonitoring` guard, so calling it again after stopping is fine, but there's no way to clear `lastActivityTimestamp` or `isIdle` without calling private methods

## Expected Behavior

- The monitor should provide a way to reset its state for daily rollover
- The singleton pattern should not prevent testing; an `ActivityMonitoring` protocol should be extracted
- After `stopMonitoring()` and `startMonitoring()`, state should be fresh

## Actual Behavior

- No `reset()` or daily rollover mechanism exists
- The `static let shared` singleton pattern prevents injecting test doubles
- `startMonitoring()` does reset `lastActivityTimestamp` and `isIdle`, but only when `!isMonitoring` - calling it while already monitoring does nothing
- The `idleCheckTask` from a previous `startMonitoring()` call continues running even if the actor's reference count drops, because `Task` captures `[weak self]` but the singleton is never deallocated

## Impact

- No daily rollover: the app accumulates state indefinitely if left running
- Untestable: cannot inject mock ActivityMonitor into BreakScheduler
- Memory: idle check tasks are never cleaned up if monitoring is started but the singleton lives forever

## Suggested Fix

1. Extract an `ActivityMonitoring` protocol
2. Add a `reset()` method for daily rollover
3. Consider making it non-singleton for testability, using dependency injection instead:

```swift
protocol ActivityMonitoring: Actor {
    var isIdle: Bool { get }
    var timeSinceLastActivity: TimeInterval { get }
    func startMonitoring()
    func stopMonitoring()
    func recordActivity()
}
```
