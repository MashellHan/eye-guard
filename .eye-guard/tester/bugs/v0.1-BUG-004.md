# BUG-004: `calculateScreenTimeScore` arithmetic error at boundary

**Bug ID**: v0.1-BUG-004
**Title**: Screen time score calculation produces incorrect values at the recommended max boundary
**Severity**: MEDIUM
**Component**: HealthScoreCalculator
**File**: `EyeGuard/Sources/Reporting/HealthScoreCalculator.swift`, lines 75-92

## Steps to Reproduce

1. Calculate screen time score with `totalTime` exactly equal to `recommendedMaxScreenTime` (8 hours = 28800 seconds)
2. The code enters the `totalTime <= recommended` branch
3. Calculate: `ratio = 1.0 - (28800 - 14400) / 14400 = 1.0 - 1.0 = 0.0`
4. Calculate: `score = Int(0.0 * 20 * 0.5) + Int(20 * 0.5) = 0 + 10 = 10`

Now test with `totalTime` slightly above recommended (e.g., 28801 seconds):
5. The code enters the `totalTime > recommended` branch
6. Calculate: `overRatio = min(28801 / 28800, 2.0) = 1.0000347`
7. Calculate: `score = 20 * max(0, 1.0 - (0.0000347) * 2.0) = 20 * 0.99993 = 19.99`
8. Result: `Int(19.99) = 19`

## Expected Behavior

The score should decrease continuously. At exactly the recommended max, both branches should produce the same score to avoid a discontinuity.

## Actual Behavior

At exactly 8 hours: score = **10** (from the `<= recommended` branch)
At 8 hours + 1 second: score = **19** (from the `> recommended` branch)

There is a **discontinuous jump from 10 to 19** when crossing the 8-hour boundary. The score actually *increases* by going from "under" to "over" the recommended max.

The issue is in the proportional calculation in the middle tier:

```swift
} else if totalTime <= recommended {
    let ratio = 1.0 - (totalTime - recommended * 0.5) / (recommended * 0.5)
    return Int(ratio * maxPoints * 0.5) + Int(maxPoints * 0.5)
```

At `totalTime == recommended`: ratio = 0.0, so the score is `0 + 10 = 10`.

But the over-recommended branch starts scoring from near-20:

```swift
} else {
    let overRatio = min(totalTime / recommended, 2.0)
    let score = maxPoints * max(0, 1.0 - (overRatio - 1.0) * 2.0)
```

At `overRatio` barely above 1.0, the score is nearly `maxPoints` (20).

## Impact

- Users at exactly 8 hours of screen time get a misleadingly low score (10/20)
- Users slightly over 8 hours get a much higher score (19/20)
- This creates a perverse incentive: going slightly over the recommended max improves your score

## Suggested Fix

Ensure the over-recommended branch starts from the same score the under-recommended branch ends at (10/20), or redesign the scoring curve to be monotonically decreasing:

```swift
private func calculateScreenTimeScore(_ totalTime: TimeInterval) -> Int {
    let recommended = EyeGuardConstants.recommendedMaxScreenTime
    let maxPoints = Double(EyeGuardConstants.screenTimeMaxPoints)

    if totalTime <= recommended * 0.5 {
        return EyeGuardConstants.screenTimeMaxPoints
    } else if totalTime <= recommended {
        // Linear from 20 -> 10 as usage goes from 50% -> 100% of max
        let ratio = (totalTime - recommended * 0.5) / (recommended * 0.5)
        return Int(maxPoints * (1.0 - ratio * 0.5))
    } else {
        // Linear from 10 -> 0 as usage goes from 100% -> 200% of max
        let overRatio = min((totalTime - recommended) / recommended, 1.0)
        return Int(maxPoints * 0.5 * (1.0 - overRatio))
    }
}
```
