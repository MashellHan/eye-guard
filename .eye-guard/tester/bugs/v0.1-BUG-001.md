# BUG-001: `resetTimersAfterBreak` is a no-op switch statement

**Bug ID**: v0.1-BUG-001
**Title**: `resetTimersAfterBreak` switch statement has empty cases - does nothing
**Severity**: HIGH
**Component**: BreakScheduler
**File**: `EyeGuard/Sources/Scheduling/BreakScheduler.swift`, lines 202-217

## Steps to Reproduce

1. Start the app and let 60 minutes pass (macro break triggers)
2. Take the macro break via `takeBreakNow(.macro)`
3. Observe that `resetTimersAfterBreak(.macro)` is called
4. Inspect the switch statement - all three cases (`micro`, `macro`, `mandatory`) contain only `break`

## Expected Behavior

When a macro break is taken, it should reset the micro-break timer independently. When a mandatory break is taken, it should reset both micro and macro timers. The break hierarchy should be:
- Mandatory break resets -> macro timer + micro timer
- Macro break resets -> micro timer
- Micro break resets -> only micro timer

After any break, specific timers should reset without resetting the entire session to zero (which loses track of how long the user has been working for longer-interval breaks).

## Actual Behavior

The switch statement is completely empty:

```swift
switch type {
case .micro:
    break       // <-- does nothing
case .macro:
    break       // <-- does nothing
case .mandatory:
    break       // <-- does nothing
}
```

After the switch, the method always resets the entire session to zero (`sessionStartTime = .now`, `currentSessionDuration = 0`), regardless of break type. This means:

- Taking a micro break resets the entire session, losing track of macro/mandatory interval progress
- All break types behave identically - no hierarchical timer management
- A user who takes micro breaks every 20 minutes will never trigger macro or mandatory breaks because the session always resets to 0

## Impact

This bug fundamentally breaks the hierarchical break scheduling model. The 20-20-20 micro breaks will prevent macro (60 min) and mandatory (2 hr) breaks from ever triggering, since the session counter resets to zero after every micro break.

## Suggested Fix

Implement per-break-type timer tracking instead of a single `sessionStartTime`. Track each break type's last-triggered time independently:

```swift
private var lastMicroBreakTime: Date = .now
private var lastMacroBreakTime: Date = .now
private var lastMandatoryBreakTime: Date = .now

private func resetTimersAfterBreak(_ type: BreakType) {
    let now = Date.now
    switch type {
    case .mandatory:
        lastMandatoryBreakTime = now
        lastMacroBreakTime = now
        lastMicroBreakTime = now
    case .macro:
        lastMacroBreakTime = now
        lastMicroBreakTime = now
    case .micro:
        lastMicroBreakTime = now
    }
}
```
