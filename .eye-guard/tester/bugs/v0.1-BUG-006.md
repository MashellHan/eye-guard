# BUG-006: `NotificationManager.snooze()` does not re-schedule notification

**Bug ID**: v0.1-BUG-006
**Title**: Snooze action cancels notification but never reschedules it
**Severity**: MEDIUM
**Component**: NotificationManager
**File**: `EyeGuard/Sources/Notifications/NotificationManager.swift`, lines 86-91

## Steps to Reproduce

1. A break notification appears (Tier 1)
2. User snoozes the notification
3. `snooze()` is called:
   - `cancelEscalation()` cancels the escalation Task
   - `dismissAllOverlays()` removes delivered notifications
   - `isNotificationActive = false`
4. No new notification is scheduled
5. The user never receives another reminder for this break

## Expected Behavior

After snoozing:
1. The current notification should be dismissed
2. A new notification should be scheduled after `EyeGuardConstants.maxSnoozeDuration` (5 minutes)
3. The rescheduled notification should restart the escalation chain

## Actual Behavior

```swift
func snooze() {
    cancelEscalation()
    dismissAllOverlays()
    isNotificationActive = false
    // TODO: Re-schedule notification after snooze duration  <-- Never implemented
}
```

The `TODO` comment confirms this is known but unimplemented. The snooze function effectively acts as a "dismiss permanently" action.

## Impact

- Users who snooze are never reminded to take their break
- The snooze feature is functionally identical to dismissing/skipping the notification
- This undermines the purpose of the escalation system

## Suggested Fix

```swift
func snooze() {
    cancelEscalation()
    dismissAllOverlays()
    isNotificationActive = false

    // Reschedule after snooze duration
    Task {
        try? await Task.sleep(for: .seconds(EyeGuardConstants.maxSnoozeDuration))
        guard !Task.isCancelled else { return }
        await MainActor.run {
            // Re-trigger with the same break type
            self.notify(breakType: lastBreakType, onTaken: lastOnTaken, onSkipped: lastOnSkipped)
        }
    }
}
```

This requires storing `lastBreakType`, `lastOnTaken`, and `lastOnSkipped` when `notify()` is called.
