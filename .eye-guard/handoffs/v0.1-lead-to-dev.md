## Summary

v0.1 delivers a **clean structural foundation** — correct module layout, good Swift 6 concurrency model, solid model layer, and working health score calculation. Score: **7/10**.

The primary gap is **integration**: the three core services (`ActivityMonitor`, `BreakScheduler`, `NotificationManager`) each work in isolation but are not wired together. The app's core value loop — detect activity → schedule breaks → notify user → record outcome — does not function end-to-end. Additionally, CGEventTap is a placeholder and the service layer has zero test coverage.

---

## Action Items for v0.2

Listed in **priority order** — complete items top-down. Items 1-4 are blockers for any user-facing functionality.

### 1. Wire ActivityMonitor → BreakScheduler (CRITICAL)

**Issue**: H1 — Components are disconnected.

- `ActivityMonitor` changes `isIdle` but nothing observes it
- `BreakScheduler.handleIdleDetected()` and `handleActivityResumed()` are never called
- **Action**: Add an `AsyncStream<Bool>` in `ActivityMonitor` that publishes idle state changes. In `BreakScheduler`, subscribe to this stream on init and call the idle/resume handlers.
- **Alternative**: Poll `ActivityMonitor.shared.isIdle` on each `tick()` in `BreakScheduler`.

### 2. Wire BreakScheduler → NotificationManager (CRITICAL)

**Issue**: H2 — `triggerBreakNotification()` only prints.

- **Action**: Replace the print statement with:
  ```swift
  NotificationManager.shared.notify(
      breakType: type,
      onTaken: { [weak self] in self?.takeBreakNow(type) },
      onSkipped: { [weak self] in self?.skipBreak(type) }
  )
  ```
- **Dependency**: Also fix H4 — store and invoke the `onTaken`/`onSkipped` callbacks in `NotificationManager.acknowledgeBreak()` and `snooze()`.

### 3. Fix Break Timer State Machine (HIGH)

**Issue**: H5 — `resetTimersAfterBreak()` resets everything regardless of break type.

- **Action**: Track per-break-type cycle counts or last-fired timestamps. On micro break: only reset micro timer. On macro break: reset macro + micro. On mandatory: reset all three.
- **Key insight**: Replace the single `sessionStartTime` with a per-type start time dictionary or compute remaining time per type from the session start.

### 4. Guard Against Duplicate Break Notifications (MEDIUM)

**Issue**: M5 — `checkForDueBreaks()` can fire multiple times per cycle.

- **Action**: Add a `lastNotifiedCycle: [BreakType: Int]` dictionary. Compute the current cycle index as `Int(elapsed / interval)`. Only fire if the cycle index advanced since last notification.

### 5. Define Protocols + Write Service Tests (HIGH)

**Issue**: H6 — Zero service-layer test coverage, no protocols for DI.

- **Action**:
  1. Define `ActivityMonitoring` protocol extracted from `ActivityMonitor`
  2. Define `NotificationSending` protocol extracted from `NotificationManager`
  3. Inject these into `BreakScheduler` via init parameters with defaults
  4. Write `BreakScheduler` tests with mock monitor and mock notifications
  5. Write `DailyReportGenerator` tests verifying Markdown output
  6. Target: 80%+ overall coverage

### 6. Replace `print()` with `os.Logger` (MEDIUM)

**Issue**: M1 — All logging uses `print()`.

- **Action**: Create a `Logger` extension in `Utils/`:
  ```swift
  import os
  extension Logger {
      static let activityMonitor = Logger(subsystem: "com.eyeguard", category: "ActivityMonitor")
      static let scheduler = Logger(subsystem: "com.eyeguard", category: "BreakScheduler")
      static let notifications = Logger(subsystem: "com.eyeguard", category: "Notifications")
      static let reporting = Logger(subsystem: "com.eyeguard", category: "Reporting")
  }
  ```
- Replace all `print("[Module]` calls with appropriate `Logger.module.info/debug/error`.

### 7. Fix BreakScheduler Timer Task Lifecycle (HIGH)

**Issue**: C1 — Misleading `[weak self]` + redundant `MainActor.run` in timer loop.

- **Action**: Simplify `startTimerLoop()`:
  ```swift
  timerTask = Task {
      while !Task.isCancelled {
          try? await Task.sleep(for: .seconds(1))
          tick()
      }
  }
  ```
- Add a `deinit` comment explaining that the Task keeps self alive and must be cancelled externally, or implement a `stop()` method.

### 8. Make UsageSession Fully Immutable (MEDIUM)

**Issue**: M6 — `var` properties undermine immutable pattern.

- **Action**: Change `endTime`, `activeTime`, `breaks` to `let`. All mutations go through functional update methods (`addingBreak()`, `ending()`). Add an `updatingActiveTime(_:)` method if needed.

### 9. Async File I/O for Report Generator (MEDIUM)

**Issue**: M2 — Synchronous file I/O called from `@MainActor` context.

- **Action**: Make `saveMarkdownReport` async. Wrap file operations in a detached task or use a background actor.

### 10. Fix NotificationManager Permission Timing (MEDIUM)

**Issue**: M3 — Permission dialog triggered on singleton access.

- **Action**: Remove `requestNotificationPermission()` from `init()`. Add a public `requestPermission()` method. Call it from `AppDelegate.applicationDidFinishLaunching()`.
